The MCP server at `apps/mcp` exposes Mbrain memory operations over stdio. Use
it as one adapter path for the Mbrain Company Brain framework.

MCP clients should be read-first. They can search memory and build context
bundles freely, but should write memory only when the user or workflow
explicitly asks to remember, update, or forget something.

## Starting the MCP server

```bash
cd apps/mcp
MBRAIN_API_URL=http://127.0.0.1:3847 \
MBRAIN_API_KEY=local-dev-secret \
bun run start
```

The server communicates over stdin/stdout using the MCP protocol.

## Primary operations

The MCP server exposes tools for recall, explicit writes, lifecycle changes,
status checks, traces, and evaluation. These are the primary framework-facing
tools, not the full server inventory:

| Tool | Description |
|------|-------------|
| `mbrain_search` | Search memory with hybrid vector + full-text |
| `mbrain_search_detailed` | Search with plans, passes, metadata, and evidence |
| `mbrain_search_kb` | Search imported knowledge base content |
| `mbrain_read_file` | Reopen a memory locator or file path |
| `mbrain_build_context_bundle` | Build a prompt-ready context bundle |
| `mbrain_add` | Add content to memory |
| `mbrain_write_event` | Write a conversation event |
| `mbrain_write_structured` | Write a stable structured memory |
| `mbrain_write_procedure` | Write a procedure or playbook memory |
| `mbrain_profile` | Get a synthesized user profile |
| `mbrain_memory_feedback` | Confirm, correct, or reject memory relevance |
| `mbrain_memory_get` | Get a structured or procedure memory |
| `mbrain_memory_update` | Update structured or procedure memory |
| `mbrain_memory_delete` | Delete or invalidate structured or procedure memory |
| `mbrain_memory_history` | Inspect structured or procedure memory history |
| `mbrain_chain_trace` | Inspect memory chain provenance |
| `mbrain_status` | Check memory provider health |

The server also includes admin, probe, benchmark, import, detailed status, and
job tools for operator workflows. Keep agent instructions focused on the
smallest tool set required for the task.

## Client guidance

Add a durable rule to the client that uses this MCP server:

```md
Use Mbrain as a read-first Company Brain. Search memory or build a context
bundle before answering when durable company, customer, user, project, or
workflow context would help. Do not write memory automatically. Write only when
the user or app explicitly asks to remember, update, or forget something.
```

For client-specific snippets, see [Adapter Contract](/guides/adapters).
