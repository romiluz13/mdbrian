Mbrain is a Turborepo monorepo built around a small supported core. The
Company Brain framework runtime is `apps/api` -> `packages/memory-bridge` ->
`packages/memory-engine`.

## Packages

| Package | Role |
|---------|------|
| `@mbrain/memory-engine` | Core MongoDB memory: embeddings, graph, episodes, search, KB, analytics |
| `@mbrain/memory-bridge` | Stable facade for the engine -- config resolution, bridge API |
| `@mbrain/memory` | Published re-export of the engine + bridge |
| `@mbrain/client` | TypeScript HTTP client SDK |
| `@mbrain/tools` | AI SDK tool helpers (Vercel AI SDK compatible) |
| `@mbrain/lib` | Shared types and utilities (internal) |

## Framework surfaces

| Surface | Role |
|---|---|
| HTTP API | App and service integration |
| TypeScript client | Typed app access to API operations |
| AI SDK tools | Explicit function tools plus optional middleware for app frameworks |
| MCP server | Agent and IDE client integration |
| Web console | Operator review and inspection |

The `@mbrain/tools` middleware helpers can write conversation events
automatically after model calls. Use explicit function tools or the TypeScript
client when an app needs strict read-first behavior.

## Apps

| App | Port | Description |
|-----|------|-------------|
| `apps/api` | 3847 | HTTP API server (Hono) |
| `apps/mcp` | stdio | MCP server wrapping the HTTP API |
| `apps/web` | 3040 | Next.js web console |
| `apps/docs` | 3003 | Mintlify docs for the supported product core |

## Status by surface

| Status | Surfaces |
|--------|----------|
| Supported public | `apps/api`, `apps/mcp`, `apps/web`, `apps/docs`, `packages/memory-engine`, `packages/memory-bridge`, `packages/mbrain-memory`, `packages/client`, `packages/tools` |
| Runtime support | `packages/lib`, `docker/mongodb`, repo-owned validation scripts |
| Not shipped | Historical research, diagnostic runs, raw artifacts, and planning notes |

## Data flow

```text
Client SDK / HTTP  ->  apps/api  ->  memory-bridge  ->  memory-engine  ->  MongoDB
                                                                                   |
MCP client (stdio) ->  apps/mcp  ->  client SDK  ----------------------------------+
```

The engine writes to MongoDB collections with automated vector indexes, text
indexes, and TTL policies. Search goes through the supported hybrid retrieval
pipeline and falls back only when the engine explicitly chooses to do so.

The first framework slice does not change this topology. It defines taxonomy,
scope policy, adapter guidance, and write safety on top of the existing runtime.
