Mbrain is a **MongoDB-native Company Brain memory framework** for AI apps,
agents, and teams.

It gives AI systems a durable memory layer backed by MongoDB collections,
MongoDB Search, Vector Search, and hybrid retrieval. The public repo focuses on
the supported product path: API server, MCP server, TypeScript client, AI SDK
tools, web console, Docker setup, framework docs, and release checks.

## Start with one loop

The fastest way to understand Mbrain is:

1. Start MongoDB.
2. Start the Mbrain API.
3. Add one memory.
4. Search for it.

## What it stores

| Memory type | Use case |
|---|---|
| Episodic events | Long-running chat, task, ticket, and workflow history |
| Semantic facts | Preferences, profile details, decisions, and current state |
| Procedural playbooks | Workflows, runbooks, and operating instructions |
| Workspace knowledge | Documents, project context, and reference material |
| Provenance | Source context, traces, citations, and lifecycle evidence |

## Runtime shape

```text
App / Agent / MCP client
  -> Mbrain API or TypeScript client
  -> Memory bridge
  -> MongoDB memory engine
  -> MongoDB
```

Mbrain keeps the integration surface small while the engine handles indexing,
retrieval, telemetry, graph relationships, and memory lifecycle behavior.

## Framework safety

Mbrain integrations are read by default. They can search memory and build
context bundles without changing stored state. Writes require explicit user,
app, operator, test, or import intent.

## Why MongoDB

Mbrain uses MongoDB as the memory substrate instead of bolting together a
separate vector database, relational store, and event log. That keeps source
documents, vector search, lexical search, metadata, provenance, and operational
queries in one place.

## Evaluation posture

Benchmark evidence is scoped by lane. Mbrain currently publishes selected
MemPalace retrieval-lane evidence only. Broader ecosystem and judged-answer
benchmarks remain under audit, and no Mem0 LongMemEval win is claimed.
